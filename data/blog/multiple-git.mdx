## **How to Tackle These Issues**

### **1. Use Git Configurations**

To manage different accounts, you can set up Git configurations that work for each repository:

**Step 1: Set Up Default Git Settings**

```bash:gitConfig
git config --global user.name "Your Default Name"
git config --global user.email "your-default-email@example.com"
```

**Step 2: Customize Settings for Specific Repositories**

```bash:local-repo
cd path/to/your/repo
git config user.name "Your Other Name"
git config user.email "your-other-email@example.com"
```

These settings will override the global defaults just for this repository.

![Git](/static/art/1.png)

### **2. Manage SSH Keys**

To handle multiple SSH keys:

**Step 1: Generate SSH Keys**

```bash
ssh-keygen -t rsa -C "your-email@example.com" -f ~/.ssh/id_rsa_account1
ssh-keygen -t rsa -C "your-other-email@example.com" -f ~/.ssh/id_rsa_account2
```

**Step 2: Add Keys to GitHub**

```bash
cat ~/.ssh/id_rsa_account1.pub
```

**Step 3: Configure SSH for Multiple Accounts**

```bash:multiple-accounts
Host github.com
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_rsa_account1
# Second GitHub account
Host github-account2
  HostName github.com
  User git
  IdentityFile ~/.ssh/id_rsa_account2
```
